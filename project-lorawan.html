LoRaWAN Asset Tracker

Long-range IoT tracking solution for industrial assets

Dec 2024 - Jan 2025 LoRaWAN, Embedded Systems, IoT

Project Overview

The LoRaWAN Asset Tracker is an innovative IoT solution designed to monitor valuable equipment across large industrial facilities with exceptional battery life and reliable long-range communication.

Key Innovations

  • Extended Range: 5km+ in urban environments using LoRa technology
  • Ultra-Low Power: 12-18 month battery life with optimized power management
  • Hybrid Positioning: Combines GPS with BLE beacons for indoor/outdoor tracking
  • Robust Design: Industrial-grade enclosure rated for -40°C to +85°C operation
  • Smart Alerts: Real-time tamper detection and geofencing notifications

Power Specifications

  • Sleep Current: 1.8µA
  • Active Current: 22mA
  • Battery: 3000mAh Li-SOCl₂
  • Voltage Range: 2.1V - 3.6V

Wireless Specifications

  • Frequency: 868MHz (EU) / 915MHz (US)
  • Max Output Power: +20dBm
  • Sensitivity: -148dBm
  • Modulation: LoRa™

Hardware Specifications

  • MCU: STM32L072CZ (ARM Cortex-M0+)
  • Memory: 192KB Flash, 20KB RAM
  • Sensors: 3-axis accelerometer
  • Enclosure: IP67 rated

Technical Implementation

Hardware Design

The custom PCB was designed for ultra-low power operation while maintaining robust performance in industrial environments:

  • 4-layer PCB with dedicated power planes
  • STM32L072CZ microcontroller with ARM Cortex-M0+ core
  • Semtech SX1276 LoRa transceiver for long-range communication
  • LIS2DH12 accelerometer for motion detection
  • IP67 rated polycarbonate enclosure

Firmware Architecture

The firmware was built on Zephyr RTOS to leverage its power management capabilities:

  • Event-driven architecture with power-aware scheduling
  • Adaptive transmission intervals (1min to 1hr) based on motion
  • Secure OTA updates with rollback capability
  • AES-128 encrypted communication
  • Watchdog timer and brown-out detection

Cloud Integration

The cloud platform provides real-time monitoring and historical data analysis:

  • The Things Network for LoRaWAN infrastructure
  • Node-RED flows for data processing and alerting
  • MongoDB time-series collections for efficient storage
  • React-based dashboard with map integration
  • REST API for integration with enterprise systems

Key Power Saving Techniques:

Intelligent Duty Cycling

Implemented a state machine that dynamically adjusts device behavior based on environmental conditions:

// Adaptive transmission algorithm
void update_operation_mode() {
    if (accelerometer.detect_motion()) {
        set_active_mode();
        gps_enable();
        transmit_interval = MIN_INTERVAL;
    } 
    else if (battery_voltage < LOW_BATTERY_THRESHOLD) {
        set_power_save_mode();
        transmit_interval = MAX_INTERVAL;
    } 
    else {
        set_normal_mode();
        transmit_interval = DEFAULT_INTERVAL;
    }
}

Aggressive Power Gating

Designed hardware with MOSFET switches to completely disconnect power from unused peripherals:

  • GPS module powered only during position fixes
  • Sensors sampled in burst mode then powered down
  • Voltage regulators with <1µA quiescent current

Clock Management

Implemented dynamic clock scaling to minimize power during processing:

  • 32MHz during RF transmission
  • 4MHz during sensor sampling
  • 32kHz in sleep mode

Why LoRaWAN Excels for Industrial Tracking

For industrial asset tracking applications, LoRaWAN provides the optimal balance of range, battery life, and cost-effectiveness. While cellular solutions offer better range, their power consumption and recurring costs make them impractical for large-scale deployments. Bluetooth and WiFi solutions lack the necessary range for most industrial facilities.

Our implementation further enhances standard LoRaWAN with adaptive power management and hybrid positioning to address common limitations of the technology.

Challenges & Solutions

Challenge: GPS Acquisition in Urban Canyons

Initial field tests showed poor GPS performance in dense urban environments with tall buildings.

Solution:

  • Implemented assisted-GPS using LoRaWAN time sync
  • Added BLE beacon support for indoor positioning
  • Developed hybrid positioning algorithm

Challenge: RF Interference in Industrial Settings

Some facilities showed packet loss due to heavy machinery causing RF interference.

Solution:

  • Implemented adaptive data rate (ADR) algorithm
  • Added frequency agility to avoid congested channels
  • Increased spreading factor in noisy environments
