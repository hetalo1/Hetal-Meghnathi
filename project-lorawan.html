LoRaWAN Asset Tracker

Long-range IoT tracking solution for industrial assets

Aug 2023 - Jan 2025 LoRaWAN, Embedded Systems, IoT Private Repository

Project Overview

The LoRaWAN Asset Tracker is a low-power, long-range tracking solution designed for monitoring valuable equipment across large industrial facilities. The system provides real-time location data with battery life exceeding 12 months, solving the challenge of tracking assets in GPS-denied environments.

Key Features

  • 5km+ range in urban environments
  • 12-18 month battery life on a single charge
  • Hybrid positioning (GPS + BLE beacons for indoor tracking)
  • Shock/tamper detection with accelerometer alerts
  • Extreme temperature operation (-40°C to +85°C)
  • Cloud dashboard with geofencing and analytics

Technical Implementation

A robust architecture combining hardware, firmware, and cloud components

Hardware Design

  • STM32L072CZ as main microcontroller (Ultra-low-power ARM Cortex-M0+)
  • Semtech SX1276 LoRa transceiver for long-range communication
  • Custom ultra-low-power PCB design with 4-layer stackup
  • 3-axis accelerometer (LIS3DH) for motion detection
  • IP67 rated enclosure with industrial-grade components
  • Optional solar charging circuit for permanent installations

Firmware

  • Zephyr RTOS for advanced power management
  • Adaptive transmission interval (1min to 1hr) based on motion
  • Over-the-air (OTA) updates via LoRaWAN
  • Encrypted communication (AES-128) for data security
  • Custom location algorithms combining GPS, BLE RSSI, and dead reckoning
  • Battery health monitoring and prediction

Cloud Integration

  • The Things Network (TTN) for LoRaWAN infrastructure
  • Node-RED flows for data processing and alerting
  • MongoDB time-series database for efficient location storage
  • React-based dashboard with real-time maps and analytics
  • REST API for integration with existing ERP systems
  • Automated reporting and alerting via email/SMS

Adaptive Transmission Algorithm

Dynamically adjusts reporting interval based on motion detection, battery level, and network conditions:

// Adaptive transmission logic
void update_reporting_interval() {
    if (accelerometer.motion_detected()) {
        interval = MIN_INTERVAL; // 1 minute during activity
        gps_enable(true); // Wake GPS for precise location
    } 
    else if (battery.level() < LOW_BATTERY_THRESHOLD) {
        interval = MAX_INTERVAL; // 1 hour to conserve power
        gps_enable(false); // Disable GPS
    } 
    else if (network_quality < 0.7) {
        interval = DEFAULT_INTERVAL * 2; // Reduce transmission frequency
    }
    else {
        interval = DEFAULT_INTERVAL; // 15 minutes normal operation
    }
    
    lora_set_tx_interval(interval);
}

Intelligent Peripheral Management

Implemented a hierarchical power management system that completely powers down unused peripherals:

  • Hardware power switches for GPS, BLE, and sensors
  • Dynamic voltage scaling for MCU (1.8V to 3.3V)
  • Clock gating for unused peripherals
  • Selective wake-up interrupts

Energy-Efficient Location Acquisition

Developed a hybrid positioning system to minimize GPS usage:

  • BLE beacon fingerprinting for indoor locations
  • Dead reckoning using accelerometer data
  • Predictive GPS wake-up based on movement patterns
  • Assisted GPS (A-GPS) for faster satellite acquisition

Why LoRaWAN Won

For industrial asset tracking, LoRaWAN provided the optimal balance of range, battery life, and cost-effectiveness. While cellular solutions offered better range, their power consumption and recurring costs made them impractical for this application. Bluetooth and WiFi alternatives couldn't provide the necessary coverage across large facilities without extensive infrastructure.

The decision matrix prioritized:

  1. Battery life - Needed to minimize maintenance
  2. Coverage - Large industrial sites with metal structures
  3. Total cost of ownership - No recurring fees
  4. Reliability - Consistent performance in harsh environments

Challenges & Solutions

Key obstacles faced during development and how they were overcome

Power Consumption

Problem: Initial prototypes only achieved 3 months battery life

Solution: Implemented hierarchical power management and adaptive transmission algorithm

Result: 5x improvement to 15 months operation

Signal Penetration

Problem: Poor performance in metal-rich environments

Solution: Optimized antenna design and implemented signal repeating through mobile assets

Result: 95% coverage in challenging industrial sites

Indoor Positioning

Problem: GPS unusable indoors, BLE beacons unreliable

Solution: Hybrid approach combining BLE fingerprinting and dead reckoning

Result: <5m accuracy in 80% of test locations

Extreme Temperatures

Problem: Battery performance degraded in cold

Solution: Selected Li-SOCl2 chemistry with heating circuit

Result: Reliable operation from -40°C to +85°C
